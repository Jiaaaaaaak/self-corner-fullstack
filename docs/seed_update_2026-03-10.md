## 二、新增資料庫欄位

本次同步新增 4 個欄位（供未來前端改版使用，目前 API 不 expose）：

| 表 | 欄位 | 類型 | 說明 |
|---|---|---|---|
| `student_personalities` | `domain_weights` | JSONB | 領域重視度，如 `{"學業":3,"情感":2,"人際":4,"規矩":2}` |
| `student_personalities` | `personality_tags` | VARCHAR(100) | 個性標籤，如 `"防衛刺蝟型"` |
| `scenarios` | `short_desc` | VARCHAR(200) | 短說明，如 `"分組衝突摔壞實驗器材"` |
| `scenarios` | `tags` | JSONB | 適合本情境的個性標籤清單 |

## 三、執行流程

### 首次部署或更新既有環境

```bash
cd backend

# 更新 DB 欄位 + 寫入真實資料（兩步合一）
python seed_data.py
```

`seed_data.py` 內部會先執行 `migrate_db()`（新增欄位），再 UPSERT 所有情境與學生個性資料，**可重複執行，不會重複插入或覆蓋測試帳號**。

> 若只需要補欄位（不動資料），也可單獨執行：
> ```bash
> python migrate.py
> ```

### 執行後重啟服務

```bash
# 重啟 FastAPI backend（Ctrl+C 後重跑）
uvicorn main:app --reload
```

前端（Vite）與 LiveKit 不需要重啟。

## 四、驗證方式

```sql
-- 確認學生個性（應有 10 筆）
SELECT id, name, personality_tags, domain_weights FROM student_personalities ORDER BY id;

-- 確認情境（應有 16 筆）
SELECT id, title, short_desc, tags FROM scenarios ORDER BY id;
```

或直接呼叫 API：
```
GET /scenarios  → 應回傳 16 筆
```
